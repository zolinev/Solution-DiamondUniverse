## Задача
#### Проблематика:
Текущий процесс продажи алмазов в Diamond Universe предполагает проведение аукциона в очном формате с физическим присутствием покупателей в центральном офисе. Эксперты покупателей лично оценивают алмазы, после чего проводятся торги.
Пандемия сделала такой формат проведения торгов невозможным.
Необходимо создать систему онлайн продажи алмазов для Diamond Universe. Процесс должен гарантировать для Diamond Universe прозрачность и прогнозируемость всех этапов, а для покупателей - точное понимание характеристик покупаемых алмазов
#### Контекст (почему задача возникла сейчас):
 1. Влияние пандемии -  падение спроса на 30% и усложнение/невозможность очного присутствия покупателей и экспертов на торгах из-за закрытия границ
 2. Конкуренты уже запускают онлайн-продажи. 

#### Текущее состояние:
Единственный канал продажи алмазов — очные аукционы в центральном офисе
#### Целевое состояние :
Реализован цифровой канал продажи алмазов через онлайн-аукцион с удалённой оценкой и торгами

#### Ограничения:
1. Срок - первый онлайн-аукцион должен быть проведен через 2 месяца
2. Оцифровка алмазов - только у подрядчка (ИС «Лаборатория»)
3. Ограничения скоупа проекта
	1. Рассматриваем только ювелирные алмазы, мелкие - за рамками проекта. 
	2. Процессы добычи и логистики алмазов до офиса- не рассматриваем

## Описание оптимального решения:
Разделяем решение на два шага - промежуточное (2 месяца) и целевое. 
#### Промежуточное решение:
##### Принципы:
1. Не дорабатываем основные системы. Для передачи данных используем выгрузки в эксель по утвержденным шаблонам и ручной ввод в системы.
2. Нагрузку по ручным загрузкам/выгрузкам ложится на менеджеров соответствующих отделов компании. На них же и ответственность за корректность данных. Для надежности на критических этапах используем подтверждение второй рукой
#### Описание:
 1. Создаем эксель-шаблон, куда менеджер офиса вносит идентификаторы и характеристики камней, подпадающих под онлайн аукцион. Помимо этого данные о таких камнях вносятся в ИС "Алмазы" так же, как и для офлайн-аукциона
 2. Отправляя алмазы в лабораторию, менеджер посылает этот эксель-файл менедежеру лобоработории и менеджеру логистики
 3. Лаборатория проводит необходимые измерения, делает фото алмазов и 3D-модель
 4. Менеджер лаборатории выкладывает фото и 3D-модель на облако, обогащает эксель-файл характеристиками алмазов и ссылками на медиа-файлы, отправляет файл менеджеру офиса + отправляет акты выполненных работ в бухгалтерию
 5. Менеджер офиса формирует эксель-файл перечнем алмазов, готовых для онлайн-продажи, их характеристиками и ссылками на медиа и отправляем его менеджеру аукциона
6. Менеджер аукциона формирует итоговый эксель-файл со списком камней и их характеристиками, которые будут продавать на ближайшем аукционе. Выкладывает этот файл и медиа на отдельное, доступное для клиентов облако
 7. Аукцион проводится либо в режиме видеоконференции, либо клиенты по почте присылают свои предложения. 
 8. Менеджер вручную вводит результаты в ИС «Аукцион»
 9. Дальше весь процесс работает аналогично офлайн-аукциону

### Целевое решение:
#### Принципы:
 1. Уходим от ручных выгрузок и вводов - минимизируем ошибки и временные затраты сотрудников
 2. Переносим хранение всех данных на сервера компании
 3.	Собираем в ИС "Алмазы" все изменения статусов алмазов из других ИС, чтобы иметь в одном ме те план/факт и историческую отчетность
#### Описание:
 1. Все данные по алмазам для онлайн-аукциона автоматически выгружаются из ИС «Алмазы» и ИС «Лаборатория»
 2. По результатам оцифровки алмазов данные автоматически передаются в ИС «Алмазы». 
 3.	По мере готовности к торгам данные о характеристиках алмазов передаются из ИС "Алмазы" и ИС "Аукцион" и из ИС "Аукцион" публикуются для клиентов
 4. Для проведения онлайн-аукциона разрабатываем для клиентов веб-интерфейс
 5. Данные, необходимые для проведения расчетов по всем операциям (оцифровка, транспортрировка, покупка) автоматически попадают в ИС «Бухгалтерия»
 6. Данные, необходимые для отслеживания статуса алмаза (отобран для аукциона, готов к транспрортировке в лабоработорию, транспонтируется в лабоработорию, принят лабораторией, оцифрован, отправлен в офис, готов к торгам, продан и тд) отправляются из профильных ИС в ИС "Алмазы"
 7. В ИС "Алмазы" создаем отчетность, отображающую план/статус по всем этапам жизненного цикла алмаза

## Что необходимо сделать:
1. Для промежуточного решения: 
	1. 2 шаблона эксель - один между менеджерами офиса, лаборатории и аукциона, второй - для клиентов
	2. Инструкции для менеджеров по работе с промежуточным решением
	3. Арендовать место в облаке
2. Для целевого
	1. Доработать ИС и интеграции (описание доработок в FIT/GAP-анализе)
	2. Инструкции для менеджеров
	3. Создать отчетность план/факт на основе данных в ИС "Алмазы" (дэшборды на Grafanа или выгрузка в файлы эксель/pdf в зависимости от требований бизнеса)
