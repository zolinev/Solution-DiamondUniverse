### FIT/GAP-анализ 

_ИС - функциональность - что нужно доработать_
1. ИС «Алмазы» (существующая) - Отбор алмазов для продажи в центральном офисе - добавить новый типа отбора «онлайн-аукцион»
2. ИС «Алмазы» (существующая) - Присвоение идентификатор алмазам - доработок не требуется
3. ИС «Алмазы» (существующая) - печать ведомостей - доработок не требуется
4. ИС «Алмазы» (существующая) - планирование работ - добавить функционал планирования работ по оцифровке, по перевозке в/из лаборатории и проведению онлайн-аукциона
5. ИС «Логистика» (существующая) - отправка алмазов клиентам - доработок не требуется
6. ИС «Логистика» (существующая) - отправка алмазов в лабораторию и обратно - создать функционал по аналогии с отправкой клиентам
7. ИС «Лаборатория» (новая) - оцифровка алмазов - реализовать новый функционал по проведению замеров алмаза (размер, чистота, флюоресценция). Должны создаваться фото и 3D-модель алмаза. Должна быть учтена привязка к идентификатору алмаза
8. ИС «Аукцион» (существующая) - публикация данных для покупателей онлайн-аукциона - реализовать модуль с веб-интерфейсом для доступа покупателей к оцифрованным данным по продающимся алмазам
9. ИС «Аукцион» (существующая) - проведение онлайн-аукциона - реализовать модуль с веб-интерфейсом для проведения онлайн-аукциона
10. ИС «Бухгалтерия» (существующая) - актирование и оплата услуг компаний-перевозчиков - доработок не требуется
11. ИС «Бухгалтерия» (существующая) - актирование и оплата услуг лаборатории - реализовать функционал по аналогии с перевозчиками
12.  ИС «Бухгалтерия» (существующая) - расчеты с покупателями - реализовать механизм автоматического выставления счетов покупателям по результатам проведения аукционов
